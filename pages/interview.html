Interview Prep

Common Questions

General

  1. Why fashion tech? What draws you to the intersection?
  2. What fashion tech product do you admire most and why?
  3. How would you improve [company's product]?
  4. What's the biggest challenge in fashion technology today?

Technical roles

  1. How would you build a recommendation system for fashion e-commerce?
  2. How do you handle cold-start in fashion recommendations?
  3. Walk through a CV pipeline for fashion product tagging
  4. How would you evaluate a virtual try-on system?

Product/Business roles

  1. How would you measure success for virtual try-on?
  2. Design a feature that reduces fashion returns
  3. What metrics matter most for fashion e-commerce?

Creative/Hybrid roles

  1. Show something you've built that bridges fashion and technology
  2. How do you balance aesthetics with technical constraints?

Case Studies

Reducing Returns

"Our fashion e-commerce has a 30% return rate. Use tech to reduce it."

  • Analyze return reasons (size/fit, color, style mismatch)
  • Propose solutions: better size rec, AR try-on, improved imagery, customer reviews
  • Estimate impact for each
  • Prioritize by effort vs impact

Trend Prediction

"Build a system to predict next season's trends."

  • Data sources: social media, runway, search data, street style
  • Pipeline: image collection, feature extraction, trend detection
  • Output: dashboards, alerts, design recommendations
  • Validation against actual trends

Personalization

"Design personalization for a luxury fashion marketplace."

  • Balance personalization vs luxury discovery
  • Data signals: browse, purchase, returns, wishlists
  • Cold-start solutions
  • Editorial curation + algorithmic recs
  • A/B testing framework

Portfolio Presentation Tips

  1. Lead with the problem, not the tech. "I wanted to solve X" not "I used PyTorch"
  2. Show visual results. Fashion is visual
  3. Quantify. "Improved accuracy by X%", "Tested with Y users"
  4. Show domain knowledge. Use correct fashion terminology
  5. 3 - 4 projects max. Quality over quantity
  6. Have a live demo. Even simple ones impress
  7. Practice the 10-minute version
